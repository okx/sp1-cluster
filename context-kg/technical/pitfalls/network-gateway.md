# Network Gateway Pitfalls

- [Pitfall] `mint_request_id` must produce exactly 32 bytes — SDK decodes via `B256::from_slice` which panics on length mismatch. Trigger: shortening or lengthening the typeid string. Correct approach: resize buffer to exactly `REQUEST_ID_LEN`; the `assert!(buf.len() <= REQUEST_ID_LEN)` enforces the upper bound. Source: `bin/network-gateway/src/ids.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] `proof_id_from_request_id` must strip trailing zeros — The 32-byte padded id round-trips on every poll; zero bytes leaking into the cluster `proof_id` won't match Postgres. Trigger: refactoring the rposition logic. Correct approach: keep the rposition over non-zero bytes; round-trip test enforces this. Source: `bin/network-gateway/src/ids.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] `request_proof` skips upload only on warm cache — Hot path checks `client.exists(&program_artifact_id, Program)` first; only on TTL miss reloads from `ProgramStore`. If Redis TTL (4h) expires between proofs AND `ProgramStore` lost the ELF, returns `FailedPrecondition`. Trigger: prove cadence > 4h with `memory` backend. Correct approach: use `fs` backend for `ProgramStore` (`GATEWAY_PROGRAM_STORE=fs`); ensure prove cadence stays within the Redis TTL window. Source: `bin/network-gateway/src/service/prover_network.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] `get_program` must return NotFound to trigger SDK `create_program` — SDK's `NetworkClient::get_program` treats ANY Ok response (even empty fields) as "already registered" and skips registration. Trigger: returning `Ok(None)`. Correct approach: always return `Status::not_found("...")` for unknown vk_hash. Source: `bin/network-gateway/src/service/prover_network.rs` + test `get_program_unknown_vk_hash_returns_not_found`. Affected module: `bin/network-gateway`.
- [Pitfall] gRPC max message size must exceed 4 MiB default — SP1 vkeys in `create_program` exceed tonic's 4 MiB default; bumped to 64 MiB on both decoding and encoding. Trigger: adding a new SP1 SDK gRPC call without setting both sizes. Correct approach: set `max_decoding_message_size` AND `max_encoding_message_size` to `MAX_GRPC_MESSAGE_SIZE = 64 MiB`. Source: `bin/network-gateway/src/lib.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] Axum `DefaultBodyLimit::disable()` is required for artifact PUT — Axum's 2 MiB default is far below realistic SP1 ELF/stdin (hundreds of MB). Trigger: re-introducing a body-limit layer. Correct approach: backpressure must come from the artifact store, not from axum. Source: `bin/network-gateway/src/artifact_http.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] `create_program` must clean up the orphan SDK-uploaded artifact — After persisting to durable `ProgramStore` + warm-keyed cluster id, the SDK-uploaded ephemeral artifact must be `try_delete`'d. Trigger: skipping the cleanup. Correct approach: keep the `client.try_delete` call after persistence. Source: `bin/network-gateway/src/service/prover_network.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] `filter` drops unsupported fields silently — `get_filtered_proof_requests` silently drops `vk_hash`/`requester`/`fulfiller`/`from`/`to`/`version`/`mode`. Trigger: callers relying on those filters. Correct approach: SDK callers must be aware; this is intentional "best-effort" behavior. Source: `bin/network-gateway/src/service/prover_network.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] `FilesystemProgramStore` writes via tmp+rename — Direct write would let a crash mid-write expose a torn record. Trigger: refactoring to a single write. Correct approach: keep `<file>.tmp` + `fs::rename` (atomic on most filesystems). Source: `bin/network-gateway/src/program_store.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] `Auth` recovers `Address::ZERO` in `None` mode — In `AuthMode::None` the requester is always the zero address regardless of signature. Trigger: dev/staging using `None` mode. Correct approach: don't treat zero address as malicious; treat as "auth disabled". Source: `bin/network-gateway/src/auth.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] `create_artifact` is signed with fixed message — The SDK literally signs `b"create_artifact"`; never change the constant without coordinating with `sp1-sdk`. Trigger: refactoring to use a domain-prefixed message. Correct approach: keep the exact bytestring. Source: `bin/network-gateway/src/service/artifact_store.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] Nonces are NOT persisted — `next_nonce` keeps an in-memory `DashMap<address, AtomicU64>`; gateway restart resets all nonces. Trigger: building replay-protection on gateway-provided nonces. Correct approach: treat nonces as opaque and process-local; don't rely on them for replay protection across restarts. Source: `bin/network-gateway/src/service/prover_network.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] Status mapping for `Cancelled` is intentional — Cluster `Cancelled` maps to SDK `Unfulfillable`/`Unexecutable` so the SDK's `wait_proof` bails terminally. Trigger: refactoring the mapping to "neutral" terminal. Correct approach: keep the explicit mapping; SDK relies on terminal-style signals. Source: `bin/network-gateway/src/status.rs`. Affected module: `bin/network-gateway`.
- [Pitfall] `CreateArtifactResponse.artifact_presigned_url` is currently the same URL as `artifact_uri` — There is no real presigning today. SDK code that distinguishes the two will silently get plain URLs. Trigger: adding real presigning. Correct approach: update both fields and likely change `Auth` to mint short-lived credentials. Source: `bin/network-gateway/src/service/artifact_store.rs`. Affected module: `bin/network-gateway`.
